# Support

This document explains how to get help with StreamBlocks.

## Getting Help

### Documentation

The best place to start is the official documentation:

- **User Guide**: https://streamblocks.hother.io
- **API Reference**: https://streamblocks.hother.io/api

### GitHub Discussions

For questions, ideas, and community support:

- [GitHub Discussions](https://github.com/hother/streamblocks/discussions)

Use Discussions for:
- Usage questions
- Feature ideas
- Sharing how you're using StreamBlocks
- General discussion

### Issue Tracker

For bug reports and feature requests:

- [GitHub Issues](https://github.com/hother/streamblocks/issues)

Before opening an issue:
1. Search existing issues to avoid duplicates
2. Use the appropriate issue template
3. Provide a minimal reproducible example for bugs

## What to Expect

### Response Times

This is a small, community-driven project. We aim to:

- Acknowledge new issues within a week
- Respond to discussions within a few days
- Review pull requests within a week

Response times may vary based on maintainer availability.

### Support Scope

We provide support for:
- The latest stable release
- The previous minor release (security fixes only)
- Issues with the library itself

We cannot provide support for:
- General Python programming questions
- Issues in third-party integrations
- Custom implementations

## Security Issues

For security vulnerabilities, please see our [Security Policy](https://github.com/hother/streamblocks/security/policy).

**Do not report security issues through GitHub Issues.**

## Contributing

If you'd like to contribute, please see our [Contributing Guidelines](.github/CONTRIBUTING.md).

## Commercial Support

There is no commercial support offering at this time. For business inquiries, contact: contact@hother.io
