## Responsibilities

Core maintainers are responsible for:

- Reviewing and merging pull requests
- Triaging issues
- Making release decisions
- Maintaining project direction and roadmap
- Enforcing the Code of Conduct

## Becoming a Maintainer

We welcome new maintainers! If you're interested in becoming a maintainer:

1. Demonstrate sustained, quality contributions to the project
2. Show good judgment in code reviews and issue discussions
3. Help other community members
4. Express your interest to an existing maintainer

## Communication

- **GitHub Issues**: For bug reports and feature requests
- **GitHub Discussions**: For questions and community discussions
- **Email**: contact@hother.io for private matters

## Decision Making

For most decisions, maintainers aim for consensus. For major decisions affecting the project direction, the lead maintainer has final say.

See [GOVERNANCE.md](GOVERNANCE.md) for more details on project governance.
